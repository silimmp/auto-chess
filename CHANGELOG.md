# 更新日志

## 2026-05-22

### 战斗演出可读性与美观度迭代

- 调整了战斗整体节奏，先整体放慢战斗前引入、普通攻击、受击停顿和清理间隔，随后又拆分出开局效果专用延迟，以及普通攻击专用延迟，让开局效果、普攻和其他战斗链路可以分别调速。
- 修正了“开局效果”提示重复出现的问题，改为只在真正触发时显示一次；同时移除了单张卡头上的重复飘字，只保留战斗顶部阶段提示和施放者高亮，降低噪音。
- 强化了攻击指向表达，区分攻击者、主目标和波及目标，并让非当前交锋单位在对应帧里短暂压暗，帮助玩家更快看清“谁在打谁”。
- 将普通攻击改成短冲刺碰撞演出：攻击者会朝目标快速前冲，在接触到目标卡面约三分之一深度时结算伤害，然后迅速回位，让“谁出手、谁被打到”比纯提示线更直观。
- 在这轮调整中逐步移除了攻击指示线以及起点、终点特效，最终只保留卡牌本体前冲、目标受击位移和伤害跳字，让战斗画面更干净，更统一地走“纯动作表现”方向。
- 战斗阶段卡面不再显示正文描述，主界面卡牌也改成默认不显示正文，只有停留在说明按钮上方时才展开描述浮层，为后续封面式卡面预留了更清爽的版面结构。
- 新增并强化了贴附在受击卡片上的伤害跳字，根据主目标和波及目标使用不同样式，并显著放大字号、提升对比度和停留时间，让玩家更容易第一眼读到本次伤害数值。
- 围绕新的短冲刺普攻演出，多次继续放慢了普通攻击专用的动作帧、命中停留、回位、拖尾与受击动画，使整段“前冲、碰撞、回位、掉血”更容易被看清。
- 清理了部分战斗文案和关键词描述，例如统一“开局效果”提示语义，去掉冗余站位文本，并移除卡牌原始描述里对“溅射”的括号解释，保持界面表达更简洁一致。
- 调整了碰撞层级规则，确保近战碰撞时攻击者始终压在受击者上方，避免上下排交叠时出现错误的前后遮挡关系。
- 修正了圣盾破裂的错误受击表现，破裂时不再额外触发对方随从抽动，只保留关键词提示，避免把护盾破裂误读成一次新的命中动作。
- 为战斗中“获得护盾”补上了独立的加护演出：目标卡牌会短暂出现罩体与柔光强化，但不再沿用受击抖动逻辑，明确区分“上盾”和“挨打”。
- 为自带护盾关键词的随从补上了常驻护罩外观，让商店、手牌与战场中的带盾单位都能一眼识别；同时多轮收敛护盾材质，去掉过强的蓝色与白色描边，只保留更柔和的发光与罩层感。
- 将玩家可见文案里的“圣盾”统一更名为“护盾”，同步更新了关键词标签、卡牌描述、开战授盾日志、破裂提示与相关说明文本，保证整套表达一致。
- 本轮改动已多次通过 `npm run smoke` 回归验证，确保战斗逻辑与视觉迭代没有破坏现有流程。

## 2026-05-23

### 关键词战斗演出重构与 PixiJS 特效层接入

- 围绕“护盾、复生、横扫、溅射”四个关键词，重新梳理了战斗帧数据结构，为 battle frame 补上 `attackKeyword`、`effectSourceId`、`hitEffects` 等字段，让主命中、波及命中、复生触发与护盾破裂都能被前端明确区分，而不再只靠通用受击状态猜测。
- 为战斗卡补上了专用关键词特效层，第一版升级了复生、横扫、溅射与护盾破裂的视觉表现：复生改成偏紫黑粒子聚拢与紫色扩散波，横扫加入白金弧形斩光与次目标依次受击，溅射加入从主目标向两侧飞散的小粒子与更偏橙色的小号伤害数字。
- 护盾视觉方向被重新推翻并收敛，不再继续沿用“整卡染蓝”或“中心脉冲”的旧方案，改为更接近能量结界的思路：战斗中优先表现卡牌外圈半透明蓝色光环、边缘缓慢环绕的蓝白粒子、边框呼吸发光，以及护盾抵挡伤害后的外扩涟漪。
- 项目已完成 PixiJS 的最小运行时接入，新增战斗特效覆盖层 `#battle-vfx-layer`，并引入本地静态 `vendor/pixi.min.js` 与 `src/vfx/battleEffects.js`。现阶段已打通 battle frame 到 Pixi canvas 的播放链路，并让护盾破裂涟漪首先改由 Pixi 绘制。
- 在继续保留现有 DOM 卡牌、数值和交互系统的前提下，开始把护盾常驻光环从 CSS 迁移到 Pixi 层：战斗区带盾卡现在会由 Pixi 运行时持续扫描并维护外圈光环、呼吸亮度与环绕粒子，为后续把受击点波纹、复生聚拢、横扫光刃和溅射粒子逐步迁入 Pixi 打好了基础。
- 继续推进关键词特效向 Pixi 覆盖层迁移：本轮把复生的粒子回拢与扩散波、横扫从攻击者经主目标向次目标延展的白金弧斩、以及溅射从主目标向两侧飞散的橙色粒子进一步接入运行时，让“谁触发、谁被波及”的空间关系更清楚。
- 对应收敛了部分 CSS 关键词贴面闪光强度，保留兜底动画但把主视觉重心让给 Pixi，减少复生、横扫、溅射在同一帧里出现双层发光和重复闪烁的问题。
- 围绕普通攻击的打击感继续多轮收敛与增强：为主命中补上了 Pixi 驱动的命中爆点、受击白闪、短暂顿帧、受击卡局部放大回弹与轻量抖镜，并反复调整命中点位置、圆核大小、冲击圈范围、火花散布半径与爆点形态，让打击重心更稳定地落在受击牌面上。
- 开始推进“战斗显示层”本身从 DOM 向 Pixi 迁移，而不再只把 Pixi 用作特效补层：新增独立的 Pixi `uiOverlayLayer`，把战斗中的伤害数字和目标高亮改为由 Pixi 直接绘制，同时停用对应的 DOM 伤害跳字输出，迈出后续战斗卡本体 Pixi 化之前的第一步。
- 为了支持新的 Pixi 静态资源，更新了 GitHub Pages 发布流程，把 `vendor/` 一并纳入部署产物，确保线上环境也能加载战斗特效层。
- 同步修正了 `npm run smoke` 的 Node 回归桩，让它跳过浏览器专属的 Pixi 运行时初始化，继续专注验证战斗、经济与回合逻辑；浏览器侧表现仍由 `npm run browser:regression` 负责覆盖。
- 随着 Pixi 战斗显示职责增加，也补齐了 smoke 测试环境里对 `querySelectorAll` 等最小 DOM 能力的桩支持，保证逻辑回归在不加载浏览器版 Pixi 运行时的前提下仍可持续验证。
- 本轮改动已通过 `npm run smoke` 与多次 `npm run browser:regression` 验证，确认 Pixi 最小接入、战斗播放链路和现有核心流程保持可用。
